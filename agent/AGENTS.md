# AGENTS 摘要（后续 Agent 快速阅读版）

## 1. 项目目标
实现“国际仓储 + 拼柜 + 结算 + 报表”的单机数据库管理系统（以可运行代码为主）。

## 2. 必读文件（按顺序）
1. 客户需求：`/agent/requirements/customer_requirements.md`
2. 开发要求：`/agent/requirements/dev_requirements.md`
3. 当前设计：`/agent/database_design.md`
4. 更新日志：`/agent/update_log.md`

补充参考：
- 导入规范：`/agent/import/`
- 导入报告：`/agent/reports/`
- 原始样本：`/2026data/`

## 3. 开发原则（必须遵守）
- 先整合“客户需求 + 开发要求”，再设计或改代码。
- 以真实数据结构为准：优先保证导入、库存状态流转、拼柜容量、结算逻辑可落地。
- 优先保证端到端闭环：登录 -> 入库 -> 拼柜 -> 结算 -> 导出 -> 备份。
- 非必要不做大范围重构；修改应小步可验证。

## 4. 关键业务约束（高优先）
- 入库主来源是“收货清单”Excel；需兼容常见字段变体。
- 体积与状态是核心字段：`IN_STOCK -> ALLOCATED -> SHIPPED`，撤回需可逆。
- 拼柜不得超容量（默认 68 CBM）。
- 结算需按系统当前规则执行（见 `update_log` 最新条目）。
- 导入失败/异常必须可追踪（批次、行级原因）。

## 5. 文档与版本要求
- 任何功能或规则变更后，必须更新：`/agent/update_log.md`
  - 写清：时间、改动内容、改动原因。
- 代码与文档统一纳入 Git 管理，提交信息需说明“做了什么 + 为什么”。

## 6. Agent 执行清单（每次任务结束前自检）
- 是否阅读了最新需求与日志。
- 是否与现有设计/实现保持一致，未破坏主流程。
- 是否补充了必要日志与文档。
- 是否完成最小验证（接口/脚本/页面至少一条可复现验证）。
