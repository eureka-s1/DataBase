# 客户收货清单（2026data）格式采样总结

采样时间：2026-04-05  
采样目录：`./2026data`（排除：`每日收货清单`、`装柜清单`、`店面对账`）  
采样文件：260 个 Excel（过滤了 `~$` 临时文件）

## 1. 总体结构结论
- 客户清单通常是“一个客户一个文件”，文件内按月份分多个 Sheet。
- 多 Sheet 是主流：
  - 260 文件中，233 文件存在明显“月份型 sheet 名”（如 `2025 7`、`2026 2`、`1月`）。
  - sheet 数量跨度较大（从 3 到 70+ 均有），说明是长期累计台账式文件。

## 2. 表头与字段（采样）
- 表头行位置以第 2 行为主（绝大多数 sheet）。
- 常见字段集合（归一后）包含：
  - `phone`（常见于第一列，纯数字）
  - `shop_no`
  - `position_or_tel`
  - `item_no`
  - `item_name_cn`
  - `material`
  - `carton_count`
  - `qty`
  - `unit_price`
  - `total_price`
  - `deposit_hint`
  - `cbm_calculated`
- 字段命名上：`位置` 与 `TEL` 都存在，但客户清单里 `TEL` 出现频率更高。
- 电话字段特征：
  - 通常位于第一列；
  - 值通常为纯数字（可能在 `.xls` 中表现为 `8092882965.0`，需归一为整数串）。

## 3. 你的补充规则（已纳入）
### 3.1 每月分 Sheet
- 采样确认成立：客户文件普遍按月建 sheet，且月度 sheet 名通常可直接识别。

### 3.2 `TEL` 列语义
- 采样确认：`TEL` 在客户清单中实际承载的是“位置/标识位”，应映射到 `position_or_tel`。
- 实务上可统一规则：`位置` 与 `TEL` 同义映射到 `position_or_tel`。

### 3.3 `... CBM FREIGHT ...` 行的业务含义
- 采样确认：大量客户文件含有类似如下行：
  - `JXLU4327901 10 CTNS 8092882965 ONWA 2026/2/7 1.7 CBM FREIGHT ...`
- 该类行为“柜次出仓/运费结算标记行”，不是普通入库货物行。
- 格式为 `柜号 + CTNS + 柜主名 + 柜主电话 + 日期 + CBM FREIGHT + 运费（人民币）+ 运费（美元）`
- 关键格式特征（需要重点识别）：
  - 柜号 token 形态建议覆盖：
    - `4~5位字母 + 6~8位数字`（如 `JXLU4327901`）；
    - `1~5位字母 + 5位及以上数字`（兼容手工变体）；
    - 纯 `6位数字` 也应作为补充识别特征（历史样本存在）。
  - 柜号不一定在首列，可能出现在行内中后段（如第 7 列附近）；
  - 行内可能出现 `CBM FREIGHT/FEIGHT`，但也可能缺失该关键字（员工填表结构不规范）。
- 业务解释（按你给的口径）：
  - 出现该行时，表示其上方一段货物已出仓；
  - 未出现该行时，上方货物默认未出仓。
- 导入建议：
  - 实操识别以“行内存在柜号（字母+数字）”为主判断条件；
  - 不再强依赖 `CBM FREIGHT/FEIGHT` 是否出现，也不强依赖柜号所在列位置；
  - 该标记行上方连续货物可标记为已出仓段，标记行下方继续新段。

### 3.5 月份 sheet 识别（用于回写）
- 月份 sheet 建议做模糊匹配，以下应视为同一个月：
  - `2026 4`
  - `2026 04`
  - `2026-4`
  - `2026 4月`
- 若已存在同月 sheet，不应重复新建。

### 3.4 电话字段口径
- 电话通常位于客户清单第一列，且为纯数字内容；
- 建议在解析时先对第一列做数字归一（去空格、去 `.0` 尾缀）后写入 `customers.phone`；
- 若同客户出现多个号码，建议按“出现频次最高”选主号码，并保留异常清单供人工复核。

## 4. 典型行形态（客户文件）
- 月度表头后是货物明细行。
- 明细区间中可能夹杂“柜次标记行 / 运费行”（含 `CBM FREIGHT` 关键字）。
- 同一客户文件可跨多月，且每月结构大体一致。

## 5. 对系统解析的建议
- 先按 sheet（月）解析，再按“是否命中柜号 token（字母+数字）”进行分段。
- 将 `TEL` 与 `位置` 统一映射到 `position_or_tel`。
- 将第一列纯数字优先识别为 `phone`，用于客户主档电话回填。
- 柜次出仓/运费结算标记行不入货物表，但可触发“上段货物已出仓”状态标注。
- 入库日期建议按批次表头识别：
  - 优先取 `SHOP NO` 表头左侧单元格（通常位于电话号码下方）；
  - 该值可能是 Excel 序列日期（如 `46056`）或日期字符串（如 `2026/2/3`）；
  - 同一 sheet 出现多个批次表头时，按段切换日期；
  - 若未识别到日期，再回退文件修改日期。
- 保留对多 sheet、大文件的性能与容错处理（read-only 流式读取优先）。

## 6. 采样说明
- 本文为结构采样总结，不替代逐客户逐月业务复核。
- 由于客户文件历史跨度长，个别 sheet 仍可能存在局部非标准格式，建议继续保持“自动探测 + 规则兜底”。
